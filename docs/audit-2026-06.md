## 1. Simulated test run — likely errors & fixes

### Environment & connectivity

| Error | When | Exact fix |
|-------|------|-----------|
| `getaddrinfo failed` on `db.*.supabase.co` | Direct Supabase host is IPv6-only on some networks | Use **Session pooler** in `DATABASE_URL`: `postgresql+asyncpg://postgres.<ref>:<pwd>@aws-1-eu-central-1.pooler.supabase.com:5432/postgres` |
| `relation "users" does not exist` | Migrations not run | `cd backend && alembic upgrade head` |
| `extension "vector" does not exist` | pgvector not enabled in Supabase | Supabase Dashboard → Database → Extensions → enable **vector**, re-run migrations |
| CORS blocked | Frontend origin not in allowlist | Set `CORS_ORIGINS=http://localhost:3010` in `backend/.env` |
| Frontend hits wrong API | Default fallback `:8000` | Set `NEXT_PUBLIC_API_URL=http://localhost:8010` in `frontend/.env.local` |
| `permission denied to set role "coord_app"` | App DB user cannot assume RLS role | Run `GRANT coord_app TO postgres;` (or dedicated app role) after migrations |
| RLS silently bypassed | `DATABASE_URL` uses Supabase `postgres` superuser | Create non-superuser app role with `GRANT coord_app`; connect as that role in prod |

### Missing API keys (degraded, not crash)

| Missing var | Symptom | Fix |
|-------------|---------|-----|
| `GEMINI_API_KEY` | Embeddings fall back to hash; LLM uses Groq/Mistral only | Add key for best semantic memory + Gemini path |
| All LLM keys empty | `"Service IA temporairement indisponible."` | Add at least `GROQ_API_KEY` |
| `S3_*` empty | Uploads return `local://` pseudo URLs; downloads empty | Configure S3-compatible storage for documents/meetings |
| `TWILIO_*` empty | WhatsApp webhook processes but no outbound messages | Add Twilio creds for WhatsApp |
| `GOOGLE_OAUTH_*` empty | Google login returns 503 | Configure OAuth or disable button in UI |

### Runtime bugs (will hit during testing)

| Bug | Symptom | Fix |
|-----|---------|-----|
| Onboarding state not refreshed | After wizard, redirect loop back to `/onboarding` | After `completeOnboarding()`, call `GET /api/auth/me` and `setUser()` in `AuthContext` |
| Wrong org slug in URL | User can open `/{other-slug}/dashboard` while JWT is different org | In `[orgSlug]/layout.tsx`, redirect if `params.orgSlug !== user.org_slug` |
| Health check path wrong in docs | Railway 404 on `/api/health` | Use `GET /health` (no `/api` prefix) or add alias route |
| Rate limit inactive | Brute-force login possible | Add `SlowAPIMiddleware` to `main.py` |
| OAuth token in URL | Token in browser history/logs | Deliver token via httpOnly cookie or fragment + immediate strip |
| Offline sync data loss | IndexedDB cleared before server confirms save | Delete local rows only after `201` + id match from `/api/sync/push` |
| `uploadFile()` no 401 retry | Meeting/doc upload fails after 15 min | Reuse refresh logic from `apiClient` in upload helper |

## 4. Mobile app strategy (June 2026)

### Options compared

| Approach | Code reuse | Offline | Push | Dev speed | Verdict for TeamBrain |
|----------|------------|---------|------|-----------|------------------------|
| **Next.js PWA** (current) | 100% web | IndexedDB (partial) | Limited (web push) | Fastest | ✓ Keep for v1 web + field agents on mobile browser |
| **Capacitor/Ionic wrapper** | ~95% of Next.js | Good (native storage plugins) | Native FCM/APNs | Fast | **Best next step** — wrap existing PWA, add push + background sync |
| **Expo (React Native)** | ~60–70% (shared TS types, API client) | Excellent | Native | Medium | Good if native UX required; rebuild UI components |
| **Flutter** | ~30% (API contracts only) | Excellent | Native | Slowest | Overkill unless dedicated mobile team |

### Recommendation

**Phase 1 (now):** Harden **Next.js PWA** — service worker caching, offline field reports, install prompt.

**Phase 2 (8–12 weeks):** **Capacitor** shell around the same frontend — native push (Firebase), geolocation, camera for field reports, background sync. ~80% code reuse with minimal rewrite.

**Phase 3 (if needed):** Expo only for modules needing heavy native UX (offline maps, voice notes for meetings).

For **real-time memory sync + offline**: add **client-side memory queue** (like field reports) syncing to `/api/sync/push` with `entity_type: memory` — not in codebase today.

## 5. Memory as core differentiator

| Capability | Status | Notes |
|------------|--------|-------|
| Persists across sessions | ✓ | PostgreSQL `memory_metadata` |
| Improves over time (compound) | ✗ | No summarization, dedup, confidence decay, or episodic→semantic promotion |
| Offline + sync | ✗ | Field reports only; memory writes are server-only |
| Sources cited on every answer | ⚠️ | Prompt asks; backfill adds sources; no hard reject if empty/wrong |
| Org-scoped reads | ✓ | `WHERE organization_id = :oid` on all memory SQL + RLS on table |
| Single write path | ✓ | `MemoryService.write_memory()` only |
| Vector search | ✓ | pgvector `<=>` with ILIKE fallback |
| Provenance in UI | ⚠️ | Kanban badges for tasks; assistant sources not clickable in UI |

**Moat assessment:** Architecture is correct (single memory layer, pgvector, module wiring). **Product moat not yet locked** — needs compounding loop, offline memory, and visible “Team Brain remembers” UX.

- [ ] Twilio webhook signature validation

- [ ] Role checks on admin/member mutations
- [ ] Audit log tenant-scoped
- [ ] Secrets never in git (`.env` gitignored ✓)
- [ ] CSP + CORS aligned with production URLs

### Hallucination guardrails (current vs needed)

| Guardrail | Current | Needed |
|-----------|---------|--------|
| Context-only answers | Prompt only | **Reject** answer if zero retrieved memories above similarity threshold |
| Source validation | Backfill from retrieval | Verify each cited `module:id` exists in context set |
| Confidence threshold | Returned but ignored | UI warning if `confidence < 0.6`; refuse to auto-create tasks |
| Empty context | Still calls LLM | Return fixed “Je n'ai pas assez d'information dans votre organisation.” |
| Structured output | JSON parse + fallback | JSON schema validation; retry once |
| Agent logging | `agent_runs` table exists | Persist `sources_used`, retrieved memory IDs, model version |
